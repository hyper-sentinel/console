# Sentinel Frontend Design Skill

This skill guides creation of all frontend interfaces for the **Sentinel AI Trading Terminal**. Every component, page, and pane must feel like it belongs in a professional trading platform that competes with Hyperliquid, Axiom.trade, and TradingView.

## Design Thinking — Sentinel's Aesthetic DNA

Before writing ANY frontend code, internalize these:

- **Purpose**: Professional multi-venue AI trading terminal. Users are crypto traders, quant devs, and power users. They expect information density, speed, and precision.
- **Tone**: **Retro-futuristic dark terminal** — think Bloomberg Terminal meets Tron Legacy. Neon accents on matte black. Data-dense but never cluttered. The feeling of piloting a spacecraft's control panel.
- **Differentiation**: This is NOT a fintech marketing page. It's a WEAPON. Every pixel serves the trader. Glassmorphism panels float over a void-black background. Cyan scan lines pulse through borders. Numbers glow.
- **Competitors to study**: Hyperliquid UI, Aster DEX, Axiom.trade, TradingView dark mode, Pump.fun

### The One Rule
**If it looks like it was generated by a generic AI assistant, it's wrong.** Sentinel has a specific identity — dark, precise, glowing, dangerous. Every surface should feel like instrumentation.

## Typography

### Fonts (loaded via Google Fonts or `next/font`)
- **Display / Headlines**: `'JetBrains Mono'` — bold, wide-tracked, uppercase for section titles
- **Body / UI Labels**: `'Inter'` weight 400-600 — clean, readable at small sizes
- **Data / Numbers / Prices**: `'JetBrains Mono'` — monospaced for alignment, tabular numbers enabled
- **Accents / Callouts**: `'Inter'` weight 700, letter-spacing 0.1em, uppercase

### Rules
- Headlines: 0.75rem–1rem, uppercase, letter-spacing 0.08em, `--text-secondary` color
- Body: 0.8125rem (13px) for dense panels, 0.875rem (14px) for standard
- Prices/Numbers: ALWAYS JetBrains Mono. Color-coded green/red for +/-.
- Never use serif fonts. Never use decorative fonts. This is a terminal.

## Color System

All colors defined as CSS custom properties in `globals.css`:

```css
:root {
  /* Backgrounds — depth layers (darkest → lightest) */
  --bg-primary:    #0A0A0A;   /* void black — page background */
  --bg-secondary:  #111111;   /* panels, cards */
  --bg-panel:      #161616;   /* elevated panels, modals */
  --bg-hover:      #1A1A1A;   /* hover states */

  /* Borders */
  --border:        #222222;   /* default */
  --border-active: #333333;   /* focused, hovered */

  /* Text */
  --text-primary:   #E5E5E5;  /* headings, important */
  --text-secondary: #999999;  /* labels, descriptions */
  --text-dim:       #555555;  /* timestamps, metadata */

  /* Accent Colors — each has a semantic meaning */
  --accent-green:   #00FF88;  /* profit, buy, success, positive change */
  --accent-red:     #FF4444;  /* loss, sell, error, negative change */
  --accent-cyan:    #00E5FF;  /* Hyperliquid venue, primary accent, links */
  --accent-purple:  #8B5CF6;  /* Aster venue, AI/agent, brand */
  --accent-yellow:  #FBBF24;  /* Polymarket venue, warnings */
  --accent-blue:    #3B82F6;  /* informational, secondary accent */

  /* Glow effects */
  --glow-green:  0 0 20px rgba(0, 255, 136, 0.15);
  --glow-cyan:   0 0 20px rgba(0, 229, 255, 0.15);
  --glow-purple: 0 0 20px rgba(139, 92, 246, 0.15);
  --glow-red:    0 0 20px rgba(255, 68, 68, 0.15);
}
```

### Venue Color Mapping
- **Hyperliquid** → `--accent-cyan` (#00E5FF)
- **Aster DEX** → `--accent-purple` (#8B5CF6)
- **Polymarket** → `--accent-yellow` (#FBBF24)
- **AI / Agent / Copilot** → `--accent-purple` (#8B5CF6)
- **CoinGecko / Market Data** → `--accent-green` (#00FF88)

### Rules
- NEVER use white backgrounds. Ever. Darkest is `#0A0A0A`, lightest panel is `#1A1A1A`.
- Accent colors are ACCENTS — used for borders, text highlights, icons. Never as fills for large areas.
- Price changes: green text for positive, red text for negative. No exceptions.
- Background gradients are subtle and radial, never linear rainbow slop.

## Motion & Animation

### Principles
- **Purposeful**: Every animation communicates state change. No animation for decoration.
- **Fast**: 150ms–250ms for micro-interactions. 300ms–500ms for page transitions.
- **Easing**: `cubic-bezier(0.16, 1, 0.3, 1)` for exits, `ease-out` for entrances.

### Required Animations
- **Page load**: Staggered fade-in from bottom (panels appear one by one, 50ms delay each)
- **Panel hover**: Border color transitions from `--border` to `--border-active`, subtle glow
- **Button hover**: Background lightens, subtle scale(1.02)
- **Data refresh**: Brief opacity pulse (1 → 0.7 → 1 over 300ms) when new data arrives
- **Price changes**: Flash green/red background for 600ms when price updates
- **Loading states**: Skeleton shimmer (dark gradient sweep), never a spinner

### CSS Classes (defined in globals.css)
```css
.animate-fade-in { animation: fadeIn 0.3s ease-out; }
.animate-slide-up { animation: slideUp 0.3s cubic-bezier(0.16, 1, 0.3, 1); }
.animate-pulse-green { animation: pulseGreen 0.6s ease-out; }
.animate-shimmer { background: linear-gradient(90deg, #161616 25%, #1a1a1a 50%, #161616 75%); background-size: 200% 100%; animation: shimmer 1.5s infinite; }
```

## Spatial Composition & Layout

### The Grid System
- Sentinel uses a **pane-based grid layout** (react-grid-layout)
- Each pane is a self-contained module with header + content
- Panes have consistent internal padding: 16px
- Pane headers: 40px height, flex row, title left, controls right

### Panel Design (`.glass-panel`)
```css
.glass-panel {
  background: rgba(22, 22, 22, 0.8);
  backdrop-filter: blur(12px);
  border: 1px solid #222;
  border-radius: 12px;
}
```

### Rules
- **Density over whitespace**: Trading terminals are dense. Don't waste space.
- **Alignment**: All numbers right-aligned. All labels left-aligned. Columns snap to grid.
- **Hierarchy**: Panel title (uppercase, dim) → content → footer/actions
- **Scroll**: Panels scroll internally, never the page. Custom thin scrollbars.

## Component Patterns

### Buttons
```

Buy/Long:             bg rgba(0,255,136,0.1), border green, text green
Sell/Short:           bg rgba(255,68,68,0.1), border red, text red
Ghost:                bg transparent, border #333, text #999, hover text white

```

### Input Fields
```
bg: rgba(255,255,255,0.04)
border: 1px solid rgba(255,255,255,0.08)
focus border: rgba(139, 92, 246, 0.5) (purple glow)
text: #E4E4E7
placeholder: #52525B
font: JetBrains Mono for numeric inputs
```

### Tables (market data, positions, orders)
```
Header: uppercase, 0.65rem, letter-spacing 0.05em, text-dim, no border
Rows: 0.8125rem, border-bottom 1px #1a1a1a
Hover: bg #1a1a1a
Alternating: NO (flat, consistent)
Numbers: right-aligned, monospace
Positive: green text
Negative: red text
```

### Status Indicators
```
Green dot:  #00FF88 — connected, active, profit
Red dot:    #FF4444 — error, disconnected, loss
Yellow dot: #FBBF24 — warning, pending
Dim dot:    #555555 — inactive, disabled
```

## What NOT to Do

- ❌ White or light backgrounds ANYWHERE
- ❌ Generic purple-gradient-on-white hero sections
- ❌ Large border-radius (>16px) on data panels
- ❌ System fonts (Arial, Helvetica) for any visible text
- ❌ Spinners (use skeleton shimmer instead)
- ❌ Centered layouts for data-dense pages (left-align everything)
- ❌ Card shadows (use border glow instead)
- ❌ Emoji as icons in the dashboard (use refined SVG or text symbols)
- ❌ Cookie-cutter SaaS landing page aesthetics
- ❌ Rounded-full pill buttons (use rounded-lg, max 8px radius)
- ❌ Any element that doesn't serve the trader's workflow

## Page-Specific Guidelines

### Login/Auth Page
- Centered card, matte black background with subtle radial purple glow
- Provider selection: grid of cards with venue colors
- Minimal, focused — get the user in fast

### Dashboard / Console (THE PRODUCT)
- Full-viewport, no scrolling page — panels fill the screen
- Sidebar (260px) + TopBar (48px) + PaneGrid (fills remaining)
- Every pane is independently scrollable
- Real-time data: prices blink on update, positions show live PnL

### Billing/Settings
- Single-column, centered content (max 640px)
- Dark cards with subtle borders
- Pricing tiers: horizontal cards, current tier highlighted with purple border

### Landing Page (if needed)
- Hero: large SENTINEL logotype, terminal aesthetic, not SaaS
- Feature sections: dark panels with cyan/purple accents
- Code snippets showing the SDK in action
- NO stock photos, NO generic illustrations
